## Governance Model

This project uses the **Tri-Perimeter Contribution Framework (TPCF)**, a graduated trust model:

### Perimeter 3: Community Sandbox
- **Access**: Open to all
- **Process**: Fork, PR, review
- **Trust Level**: None required
- **Focus**: Demonstrating competence and alignment

### Perimeter 2: Trusted Contributors
- **Access**: Direct commit rights to feature branches
- **Requirements**:
  - 5+ quality PRs merged
  - 3+ months of sustained contribution
  - Demonstrated understanding of codebase
  - Alignment with Code of Conduct
- **Responsibilities**:
  - Code review participation
  - Issue triage assistance
  - Community mentorship

### Perimeter 1: Core Team
- **Access**: Full repository rights
- **Requirements**:
  - 12+ months as Perimeter 2 contributor
  - Demonstrated long-term commitment
  - Deep codebase knowledge
  - Community trust
- **Responsibilities**:
  - Strategic direction
  - Release management
  - Security response
  - Maintainer recruitment

## Decision Making

### Consensus-Seeking
- Prefer consensus among maintainers
- Seek input from affected parties
- Document decisions in issues/ADRs

### Conflict Resolution
1. Discussion in relevant issue/PR
2. Escalation to core team if needed
3. Core team has final decision authority
4. Focus on project health over individual preferences

### Major Decisions
- Architecture changes
- Dependency additions
- Security policies
- Governance changes

**Process**: Proposal → Discussion (7 days) → Decision → Documentation

## Becoming a Maintainer

### Path to Perimeter 2 (Trusted Contributor)

1. **Contribute Quality Code**
   - Submit well-tested PRs
   - Follow coding standards
   - Respond to review feedback

2. **Engage with Community**
   - Help others in issues
   - Participate in discussions
   - Review others' PRs

3. **Demonstrate Reliability**
   - Sustained contribution (3+ months)
   - Respect Code of Conduct
   - Show long-term interest

4. **Nomination**
   - Self-nomination or nomination by existing maintainer
   - Core team votes (consensus preferred)
   - Announcement in README

### Path to Perimeter 1 (Core Team)

- Long-term commitment (12+ months)
- Demonstrated leadership
- Broad codebase knowledge
- Community trust and respect
- Invitation by existing core team
